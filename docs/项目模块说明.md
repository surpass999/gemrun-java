dependencies
作用：集中管理 Maven 依赖版本（BOM / 依赖管理）。
内容典型：父 POM 或 dependencyManagement、版本号常量、对外统一导出第三方库版本（Spring Boot、MyBatis-Plus、Lombok 等）。
影响：保证子模块使用的依赖版本一致，便于升级与安全管理。
framework
作用：后端公共框架扩展与通用基础组件。
具体实现：通用工具类、异常处理、统一响应/DTO、拦截器/过滤器、通用注解、自定义 starter 或 auto-configuration、跨模块的公共常量和基础抽象（如 BaseEntity、BaseService、BaseController）。
与安全/鉴权/序列化相关的公共配置通常也在此处提供（例如跨模块的 TokenUtils、SecurityAdapter、统一配置加载）。
module-system（或 module-system 对应的系统模块）
作用：系统核心功能模块（用户/角色/菜单/权限/部门/岗位/租户等）。
功能点：用户管理、角色管理、菜单权限、组织结构（部门）、岗位管理、登录与鉴权、在线用户、登录/操作日志、租户管理、字典/地区、错误码管理、系统配置、通知公告、应用管理（SSO）等。
角色：为其它业务模块提供统一的认证/鉴权、用户/机构/权限数据模型与 API。
module-infra（基础设施模块）
作用：平台级基础能力集合。
功能点：代码生成器（Java/Vue/SQL 等）、文件服务（本地/S3/MinIO/云存储接入）、配置管理、定时任务调度与任务日志、API 文档（Swagger / springdoc）、数据库文档（Screw）、WebSocket、消息队列工具（Redis Stream / PubSub、封装 Rabbit/Kafka）、链路/监控/日志接入适配、幂等/限流/分布式锁工具（基于 Redisson）。
影响：提升开发效率并封装运维监控接入点。
module-member（会员中心）
作用：面向 C 端/会员体系的功能模块。
功能点：会员管理、会员标签/分组、会员等级与成长值、积分签到、会员权益/规则、会员搜索与导出、会员画像支持（用于营销推送）。
与电商/运营模块集成（促销、订单关联、积分抵现等）。
module-mall（商城/电商模块）
作用：实现电商核心业务。
功能点：商品/分类/SKU 管理、库存/库存扣减策略、购物车、下单/订单管理、订单支付对接（与 module-pay 联动）、促销/优惠券、运费模板、售后/退货管理、商品上下架逻辑、商品搜索与列表。
与会员/支付/仓库/配送等模块协作。
module-pay（支付模块）
作用：支付能力封装与接入。
功能点：商户/应用信息配置、支付订单与退款订单的创建与查询、支付回调处理（异步通知）、多支付通道适配（微信/支付宝/其他）、支付接入示例、签名校验、重试/幂等处理、回调日志。
影响：为商城、账单等业务提供统一的支付接入层。
module-bpm（工作流 / BPM 模块）
作用：基于 Flowable 的工作流能力模块。
功能点：流程模型管理（BPMN）、流程设计器（可能集成在线拖拽设计或钉钉/飞书式轻设计器）、流程表单配置、流程实例/任务管理、会签/或签/委派/转办/加签/驳回/撤销/超时规则/触发器、流程监控与流程报表、历史任务与日志。
提供：流程发起、审批页渲染、任务推送与通知集成。
module-report（报表与大屏模块）
作用：可视化报表与大屏展示平台模块。
功能点：报表设计器（自定义数据报表、图形报表）、大屏拖拽设计器、图表组件集成、报表导出/打印、报表权限与数据源配置、仪表盘/监控面板支持。
与数据源/查询层和前端展示密切关联。
module-crm（CRM 模块）
作用：客户关系管理业务模块。
功能点：客户/线索/联系人 管理、商机/销售漏斗、活动/沟通记录、跟进计划、销售人员/权限分配、客户画像与统计分析、营销触发（可与短信/邮件/站内信整合）。
目标：支撑企业业务的 B2B/B2C 客户管理流程。
module-erp（ERP 模块）
作用：企业资源计划相关业务模块。
功能点：采购/销售/库存/仓库/入库出库/账务基本对象、物料编码/供应商/采购单/入库单/出库单、库存盘点、库存占用/回滚策略、报表与对账工具。
面向：企业端的供应链与库存管理需求。
module-ai（AI / 大模型模块）
作用：集成大模型与 AI 能力的模块。
功能点（推测/常见实现）：LLM 接入适配器（OpenAI、HuggingFace、国内模型等）、对话/问答接口、prompt 管理、向量/嵌入计算与向量检索（若接入向量库）、简单的 AI 应用示例（智能摘要、智能问答、文生图/文生代码等）。
影响：为业务提供智能搜索、推荐或自动化助手能力。
module-mp（微信公众号 / 小程序模块）
作用：对接微信公众号/小程序的能力模块。
功能点：公众号账号管理、多账号支持、粉丝管理（关注/取关）、消息接收/回复、模板消息、菜单管理、素材管理、消息统计、同步粉丝数据与粉丝标签操作。
与第三方平台（微信）签名校验与回调处理相关。
module-iot（物联网模块）
作用：设备接入与物联网场景支持（若存在）。
功能点（常见）：设备注册/鉴权、设备模型/属性/遥测数据接入、设备指令下发、协议适配（MQTT/HTTP）、设备在线状态监控、设备数据历史/告警、设备与用户/租户的绑定。
注意：具体实现依赖项目内该模块具体代码，但通常负责将边缘设备数据接入平台。
module-report（已列出，若有重复请参照上文）
server（或 base-server）
作用：整合启动工程（Spring Boot 启动器），包含管理后台与 APP 服务端的启动模块。
职责：Spring Boot 主类（main）、公共配置文件（application-.yml）、统一的跨模块配置（数据源、Redis、Flowable、消息队列、监控接入）、统一的异常处理、拦截器链、全局安全配置（Spring Security / Token 机制）、CORS/静态资源配置、集中化的 API 网关或微服务入口（如果采用单体则直接暴露 REST 接口）。
运行：负责将所有模块打包运行或作为部署单元。
script
作用：项目的运维/开发脚本集合。
内容：本地启动脚本、数据库初始化脚本、Dockerfile / docker-compose 启动、部署脚本、备份脚本、打包/发布脚本、CI 辅助脚本（可能有 shell / bat / SQL 等）。
便捷性：帮助开发者快速在本地或容器环境启动与初始化服务。
sql
作用：数据库层面的脚本与建表/初始化数据。
内容：建表语句、示例数据、存储过程/视图、索引建议、各模块对应的 schema 脚本、可能的迁移脚本（baseline）。
使用场景：快速复现 demo 数据库、手动建库或用于部署时初始化。
pom.xml（顶级父 POM）
作用：Maven 父构建文件，定义模块聚合、依赖管理、插件配置、构建/打包规则、模块顺序等。
影响：统一构建、模块化管理、多模块聚合打包与发布。
README.md
作用：项目总体文档、功能说明、技术栈、模块图、示例地址与快速启动指南、模块关系说明、演示截图与常见问题。
用途：开发/部署/评估项目前的入口文档；包含模块描述与示例链接（README 已列出模块对应职责）。
补充说明（公共技术栈与交互点）：
数据层：MySQL + MyBatis-Plus（CRUD/分页/Wrapper 辅助），sql 中保有表结构与初始化数据。
缓存/分布式：Redis（缓存、分布式锁、限流、消息通道等），Redisson 用于分布式原语。
工作流：Flowable 集成在 module-bpm，提供在线设计器与流程执行能力。
授权鉴权：Spring Security + Token（JWT 或自定义 Token） + Redis 存储在线信息/会话。
文档与运维：springdoc/Swagger 自动接口文档、Screw 生成数据库文档、SkyWalking/BootAdmin 等用于监控与链路追踪（在 module-infra/server 有接入点）。
单元测试：JUnit + Mockito（README 提到覆盖单元测试）。
模块划分：后端采用 Spring Boot 多模块结构，每个 module-* 为独立功能域，server 为入口，framework/dependencies 提供基础设施支持。
